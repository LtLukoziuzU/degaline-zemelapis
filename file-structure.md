# File Structure

```
degaline_project/
├── CLAUDE.md                    ← project instructions for Claude Code
├── progress.md                  ← task tracking and completed steps
├── data-flow.md                 ← data flow diagram
├── file-structure.md            ← this file
├── geocoding-strategy.md        ← Photon config, cache format, failure handling
├── xlsx-filehandling.md         ← xlsx source URL, column map, "neprekiauja" handling
├── error-handling-practices.md  ← error handling guidelines
├── dev-environment.md           ← confirmed tools installed on developer's machine
│
├── pipeline.py                  ← data pipeline (GitHub Actions + local dev)
├── server.py                    ← local dev HTTP server (port 58472)
│
├── index.html                   ← production frontend (GitHub Pages)
├── dev.html                     ← developer geocoding verification tool
│
├── .github/
│   └── workflows/
│       └── update.yml           ← daily CI/CD: run pipeline, commit data, deploy to Pages
│
├── lib/
│   ├── leaflet/
│   │   ├── leaflet.min.js
│   │   ├── leaflet.min.css
│   │   └── images/
│   └── leaflet.markercluster/
│       ├── leaflet.markercluster.js
│       ├── MarkerCluster.css
│       └── MarkerCluster.Default.css
│
└── data/
    ├── stations.json            ← generated by pipeline.py; served to browser
    ├── geocache.json            ← address → { lat, lng, source } cache
    ├── verifications.json       ← dev.html verification state (correct/incorrect per station)
    ├── company-colors.txt       ← reference color palette for companies
    ├── manual-geocache-addresses.md  ← addresses that needed manual coordinate patches
    ├── latest.xlsx              ← downloaded by pipeline.py; gitignored
    ├── logs/                    ← daily change logs written by pipeline.py
    └── history/                 ← daily JSON snapshots written by pipeline.py
```
